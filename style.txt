
general coding style in use, mostly-

    function declarations-
        will not list void in empty argument function
        void somefunc();

        argument types listed without argument names
        void somefunc(uint8_t, bool)

    common function argument names-
        e = enum
        v = value
        tf = bool

        void somefunc(MYENUM e, bool tf){ }
        void somefunc(uint8_t v){ }

    function names-
        lowercase, shorter is better
        if two words, will most likely be sparated by underscore
        if in same class, but is register specific, will sometimes
          be split into two words

        Osc::clk_src()
        Osc::pll_set()
        Osc::tun_auto()
        I2c::stat_rxfull()
        I2c:on()

    class names-
        capitalized, short is better

        I2c
        Uart
        Wdt
        Pins
        Osc


    function braces-
        void somefunc()
        {
            //code here
        }

    if/while/loops can eliminate braces if only one statement follows
        for(auto i = 0; i < 4; i++) buf[i] = 0;

    if only one statement follows, but a new line is used, then use braces
        if(i == 123 && j == 456 && k == 789 && m == 222){
            somefunc(i, k);
        }

    static class access is generally used the format-
    Reg r; Syskey sk; Irq ir;
